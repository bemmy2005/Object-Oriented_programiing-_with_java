Unit 4: Constants, Variables, Data Types

Variables in Java

A variable is a container that holds a value that can be changed during program execution.

Variable Declaration Syntax:

dataType variableName = initialValue; 
int age = 25;
double salary = 50000.50;
char grade = 'A';
boolean isEmployed = true;
String name = "John Doe";

Constants

Constants are variables whose values cannot be changed once assigned.

// Using the final keyword
final double PI = 3.14159;
final int MAX_USERS = 100;

// Naming convention: uppercase with underscores
final String DATABASE_URL = "jdbc:mysql://localhost:3306/mydb";

Variable Scope

Class/Instance Variables

Declared within a class but outside any method. Each object has its own copy.

public class Student {
    private int id;  // instance variable
    private String name;  // instance variable
}

Static/Class Variables

Shared among all instances of a class. Only one copy exists.

public class School {
    private static int studentCount;  // static variable
}

Local Variables

Declared within a method. Only accessible within that method.

public void calculateArea() {
    double radius = 5.0;  // local variable
    double area = Math.PI * radius * radius;
}

Primitive Data Types

Type Size Range Example
byte 8 bits -128 to 127 byte b = 100;
short 16 bits -32,768 to 32,767 short s = 1000;
int 32 bits -2^31 to 2^31-1 int i = 100000;
long 64 bits -2^63 to 2^63-1 long l = 100000L;
float 32 bits ±3.4e−038 to ±3.4e+038 float f = 3.14f;

boolean 1 bit true or false boolean b = true;

Reference Types

Reference types store references to objects rather than the actual values.

Classes

String name = "John";
Student s = new Student();

Arrays

int[] numbers = new int[5];
String[] names = {"Alice", "Bob"};

Type Casting

Widening (Implicit)

Converting smaller type to larger type

int i = 100;
long l = i;  // Automatic
float f = l;  // Automatic

Narrowing (Explicit)

Converting larger type to smaller type

double d = 100.04;
long l = (long)d;  // Manual
int i = (int)l;  // Manual

Best Practices

  • Use meaningful variable names that describe their purpose
  • Follow camelCase naming convention for variables (e.g., firstName)
  • Use UPPERCASE_WITH_UNDERSCORES for constants
  • Initialize variables when declaring them
  • Declare variables in the smallest scope possible
  • Use appropriate data types to save memory
  • Be careful with type casting to avoid data loss
  • Use final for values that should not change
