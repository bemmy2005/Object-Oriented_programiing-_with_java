Unit 10: Exception Handling

What are Exceptions?

Exceptions are events that occur during program execution that disrupt the normal flow of instructions.

Definition: An exception is an object that represents an error or unexpected condition.

Java uses exceptions to:

  • Separate error-handling code from regular code
  • Propagate errors up the call stack
  • Group and differentiate error types
  • Handle errors in a centralized location

Checked vs. Unchecked Exceptions

Checked Exceptions

  • Must be caught or declared
  • Compile-time checking
  • Extend Exception directly
  • Example: IOException

Unchecked Exceptions

  • Don't need to be caught
  • Runtime checking
  • Extend RuntimeException
  • Example: NullPointerException

try-catch-finally

try {
    // Code that might throw an exception
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Code to handle the exception
    System.out.println("Cannot divide by zero");
} finally {
    // Code that always executes
    System.out.println("This always runs");
}

finally block: Executes regardless of whether an exception occurs or is caught.

Multiple catch Blocks

try {
    // Code that might throw exceptions
} catch (ArithmeticException e) {
    // Handle arithmetic exceptions
} catch (NullPointerException e) {
    // Handle null pointer exceptions
} catch (Exception e) {
    // Handle all other exceptions
}

Order matters: Catch specific exceptions before general ones.

Custom Exceptions

public class InsufficientFundsException extends Exception {
    private double amount;
    
    public InsufficientFundsException(String message, double amount) {
        super(message);
        this.amount = amount;
    }
    
    public double getAmount() {
        return amount;
    }
}

throw and throws

throw

Used to explicitly throw an exception

throw new Exception("Error");

throws

Declares exceptions a method might throw

void method() throws IOException {}

Try-with-resources (Java 7+)

// Automatically closes resources that implement AutoCloseable
try (FileReader reader = new FileReader("file.txt");
     BufferedReader br = new BufferedReader(reader)) {
    
    String line = br.readLine();
    System.out.println(line);
    
    // Resources automatically closed when block exits
} catch (IOException e) {
    System.out.println("Error reading file: " + e.getMessage());
}
