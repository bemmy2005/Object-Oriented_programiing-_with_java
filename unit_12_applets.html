Unit 12: Applet Programming

Introduction to Applets

Java Applets are small programs that can be embedded in HTML pages and run in a web browser.

Important Note: Applets are deprecated in modern Java versions (since Java 9) and most browsers no longer support them. This unit is included for historical context and educational purposes.

Applets provide:

  • Platform independence
  • Security through sandbox model
  • Dynamic web content
  • Network-aware applications

Applet Lifecycle

init()

Called when applet is first loaded. Used for one-time initialization.

start()

Called after init() and each time applet is revisited in browser.

paint(Graphics g)

Called to draw content on applet. Also called when applet needs redrawing.

stop()

Called when browser leaves the page containing the applet.

destroy()

Called when browser closes the applet. Final cleanup.

Applet Parameters

Parameters can be passed to applets from HTML:

// In HTML
<param name="message" value="Hello World">
// In Java
String message = getParameter("message");

Applet Security

Applets run in a security sandbox with restrictions:

  • Cannot access local files
  • Cannot connect to servers other than the one they came from
  • Cannot start programs on the client
  • Cannot load libraries or define native methods

Note: Signed applets can request additional permissions.

Simple Applet Example

import java.applet.Applet;
import java.awt.Graphics;

public class SimpleApplet extends Applet {
    
    @Override
    public void init() {
        setBackground(java.awt.Color.white);
    }
    
    @Override
    public void paint(Graphics g) {
        g.drawString("Hello, Java Applet World!", 50, 50);
        g.drawRect(20, 70, 150, 30);
        g.drawString("I am an Applet", 40, 90);
    }
}
