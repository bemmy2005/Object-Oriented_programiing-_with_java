Unit 9: Java Packages

Introduction to Packages

Packages in Java are used to organize classes and interfaces into namespaces, providing:

  • Better organization of code
  • Prevention of naming conflicts
  • Access control between packages
  • Easier code reuse and maintenance

Package Declaration:

package com.example.myapp;

Must be the first statement in the file (before imports)

Java API Packages

Java provides many built-in packages:

java.lang

Core classes (String, Math, etc.)

Imported automatically

java.util

Collections, Date, Scanner

Must be imported

java.io

Input/output operations

Must be imported

java.net

Networking capabilities

Must be imported

Importing Classes

// Import a specific class
import java.util.ArrayList;

// Import all classes from a package
import java.util.*;

// Static import (for static members)
import static java.lang.Math.PI;

Creating Custom Packages

Steps to create and use custom packages:

  1. Create a directory structure matching the package name
  2. Add the package statement at the top of your Java files
  3. Compile and use the package

Directory Structure Example:

com/
└── mycompany/
    └── myapp/
        ├── Main.java
        ├── model/
        │   └── User.java
        └── util/
            └── Helper.java

Naming Conventions

Package names should be:

  • All lowercase
  • Unique to avoid naming conflicts
  • Typically use reverse domain name notation

Example: If your domain is example.com, your package might be com.example.project

Access Control

Package Example

// File: com/example/util/Calculator.java
package com.example.util;

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

// File: com/example/Main.java
package com.example;

import com.example.util.Calculator;

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println(calc.add(5, 3));
    }
}
