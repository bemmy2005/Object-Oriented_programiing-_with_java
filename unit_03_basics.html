Unit 3: Learning the Basics of Java

Java Program Structure

Every Java program consists of at least one class definition:

public class MyProgram {
    // Class body
    public static void main(String[] args) {
        // Main method body
        System.out.println("Hello!");
    }
}

Key Components:

  • Class Declaration: public class MyProgram
  • Main Method: Entry point for execution
  • Statements: Instructions ending with semicolons
  • Comments: Non-executable explanatory text

Java Tokens

Token Type Examples
Keywords public, class, static, void
Identifiers MyProgram, main, args
Literals "Hello!", 42, true
Operators +, -, *, /, =
Separators {}, (), ;, ,

Java Virtual Machine (JVM)

The JVM is the foundation of Java's platform independence:

Java Source Code (.java)
↓ Compiler (javac)
Bytecode (.class)
↓ JVM
Machine Code

Key Point: Java code is compiled once into bytecode, which can then run on any device with a JVM.

Command Line Arguments

Java programs can receive input through command line arguments:

public static void main(String[] args) {
    for (int i = 0; i < args.length; i++) {
        System.out.println("Argument " + i + ": " + args[i]);
    }
}

To run with arguments: java MyProgram arg1 arg2 arg3

Programming Style

Good programming style improves readability and maintainability:

  • Indentation: Use consistent indentation (typically 4 spaces)
  • Naming: Use camelCase for variables/methods, PascalCase for classes
  • Comments: Add meaningful comments to explain complex logic
  • Whitespace: Use blank lines to separate logical sections

Implementing a Java Program

  1. Write source code in a text editor (HelloWorld.java)
  2. Compile using javac HelloWorld.java to create HelloWorld.class
  3. Run using java HelloWorld (note: no .class extension when running)
